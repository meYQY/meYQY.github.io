From 0 to MVP

Sample template — replace with your own principles and examples.

Principles

  • Shortest path to value: define one core user outcome and ship that.
  • Make uncertainty visible: write assumptions, add basic metrics, review weekly.
  • Reduce complexity aggressively: narrow scope, cut features, improve defaults.

Cadence

  • Daily: one shippable change.
  • Weekly: a demo (even if rough) and a note.
  • Biweekly: reset plan based on data + conversations.
